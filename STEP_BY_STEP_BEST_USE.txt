NBA PROP ENGINE - STEP BY STEP (BEST USE)
=========================================

1) Open PowerShell and go to the project folder.
   cd "C:\Users\thegr\OneDrive\Desktop\nba data ver 2"

2) Confirm dependencies are in the project virtual environment.
   .\.venv\Scripts\python.exe -m pip install -r .\requirements.txt

3) (Optional but recommended) Set sportsbook odds API key for discrepancy checks.
   $env:ODDS_API_KEY = "YOUR_KEY_HERE"

4) Start the app with the helper script (it auto-elevates for stable NBA API access).
   powershell -ExecutionPolicy Bypass -File ".\run_ui.ps1" -OddsApiKey "YOUR_KEY_HERE"
   If you do not use odds features, you can omit -OddsApiKey.

5) Open the UI.
   http://127.0.0.1:8787

6) Use the "Slate" panel first.
   - Click "Refresh Games".
   - Pick a matchup and click "Use Home" or "Use Away" to prefill teams.

7) Run high-quality Prop EV checks.
   - Enter Player ID or Player Name, then Stat, Line, Over/Under odds.
   - Keep isHome and isB2b accurate for the game context.
   - Click "Run Prop EV".
   - Read: projection, over/under probabilities, push probability, EV%, verdict.
   - For automatic line shopping, click "Auto Sweep Best Line".
     This ranks available book lines by EV for your chosen player/stat.

   Quick player lookup from terminal:
   .\.venv\Scripts\python.exe .\nba_mod.py player_lookup "anthony edwards" 10

8) Compare sportsbook discrepancies before placing decisions.
   - In "Sportsbook Odds", set regions/markets/bookmakers.
   - Use markets like: h2h,spreads,totals.
   - Click "Load Pregame Odds" for upcoming lines.
   - Click "Load Live Odds" when games are in progress.
   - Focus on "Top Line Discrepancies" (best vs worst book and gap %).
   - For player props, use "Auto Sweep Best Line" in the Prop EV panel.

9) Use parlay analysis only after single-leg validation.
   - Validate each leg first with Prop EV.
   - Then use "Parlay EV" to evaluate correlation-adjusted joint probability and EV.

10) Track best edges and settle yesterday's picks.
    - Best edges today:
      .\.venv\Scripts\python.exe .\nba_mod.py best_today 15
    - Settle yesterday:
      .\.venv\Scripts\python.exe .\nba_mod.py settle_yesterday
    - View yesterday results:
      .\.venv\Scripts\python.exe .\nba_mod.py results_yesterday 50
    - UI equivalent:
      Use the "Tracking & Settlement" panel buttons:
      "Load Best EV", "Settle Date", and "Load Results".

11) Export settled picks for training and retrain model (advanced).
    - Export rows:
      .\.venv\Scripts\python.exe .\nba_mod.py export_training_rows .\examples\training_from_journal.csv csv
    - Build a CSV/JSON/JSONL dataset with numeric features and target "actual".
    - Example:
      .\.venv\Scripts\python.exe .\nba_mod.py train_model .\examples\calibration_sample.csv actual auto 0.5 .\models\my_model.json
    - Track RMSE and R2 after each retrain.
    - Option 1 ML gate flow:
      .\.venv\Scripts\python.exe .\nba_mod.py train_projection_ml .\examples\training_from_journal.csv actual auto 0.2 50 gradient_boosting pickDate .\models\candidate_projection_ml.pkl
      .\.venv\Scripts\python.exe .\nba_mod.py promote_projection_ml .\models\candidate_projection_ml.pkl .\models\production_projection_model.pkl 1.0 1.0 0

12) Recommended daily workflow.
    - Refresh slate.
    - Run Prop EV on shortlist.
    - Check best EV list (best_today).
    - Check pregame odds discrepancies.
    - Settle yesterday and inspect hit/ROI.
    - Export settled rows for retraining.
    - Re-check live odds if betting in-play.
    - Retrain model on a schedule (daily/weekly).
    - Optional Option 3 check:
      .\.venv\Scripts\python.exe .\nba_mod.py injury_news LAL 24

13) If something fails.
    - WinError 10013: run via run_ui.ps1 and approve admin prompt.
    - Odds endpoints error about ODDS_API_KEY: set key and relaunch.
    - No live odds events: there may be no in-play NBA games at that moment.
